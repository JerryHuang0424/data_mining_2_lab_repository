## 四、实验原理

### 4.1 问题描述

泰坦尼克号生存预测是一个典型的二分类问题，目标是根据乘客的个人信息预测其在海难中是否能够生存。这是一个经典的Kaggle竞赛数据集。

### 4.2 机器学习模型

实验对比了8种不同的机器学习算法：

1. **逻辑回归 (Logistic Regression)**: 线性分类模型，适用于线性可分问题
2. **决策树 (Decision Tree)**: 基于树结构的分类模型，易于解释
3. **随机森林 (Random Forest)**: 集成学习模型，通过多棵树提升性能
4. **K近邻 (KNN)**: 基于实例的分类方法
5. **支持向量机 (SVM)**: 寻找最优超平面的分类器
6. **朴素贝叶斯 (Naive Bayes)**: 基于贝叶斯定理的概率分类器
7. **梯度提升 (Gradient Boosting)**: 基于弱学习器的迭代优化算法
8. **AdaBoost**: 自适应提升算法

### 4.3 评估指标

- **准确率 (Accuracy)**: 预测正确的样本比例
- **精确率 (Precision)**: 预测为正类的样本中真正为正类的比例
- **召回率 (Recall)**: 实际为正类的样本中被正确预测的比例
- **F1分数**: 精确率和召回率的调和平均数

## 五、实验步骤

### 5.1 数据预处理

**数据加载**:
- 训练集: 891条记录
- 测试集: 418条记录

**缺失值处理**:
- 年龄: 用中位数填充 (177个缺失值)
- 票价: 用中位数填充 (1个缺失值)
- 登船港口: 用众数填充 (2个缺失值)

**特征工程**:
1. **FamilySize**: SibSp + Parch + 1 (家庭规模)
2. **IsAlone**: FamilySize == 1 (是否独自一人)
3. **Title**: 从姓名提取称谓 (Mr, Mrs, Miss, Master等)
4. **AgeGroup**: 年龄分段 (Child, Teen, Young Adult, Adult, Senior)
5. **FareGroup**: 票价分段 (Low, Medium, High, Very High)

**特征编码**:
- 使用LabelEncoder将分类变量转换为数值

**最终特征集**: 12个特征

### 5.2 数据分割

采用80:20的分层划分比例：
- 训练集: 712条 (80%)
- 验证集: 179条 (20%)

### 5.3 模型训练与评估

对每个模型执行以下步骤：

1. **模型初始化**: 设置随机种子确保可复现性
2. **模型训练**: 在训练集上拟合模型
3. **验证集预测**: 在验证集上评估性能
4. **交叉验证**: 5折交叉验证评估泛化能力
5. **训练时间记录**: 记录模型训练耗时

### 5.4 模型对比

- 按交叉验证准确率排序
- 绘制性能对比图表
- 分析各模型的优缺点

### 5.5 最佳模型选择与预测

1. 选择交叉验证准确率最高的模型
2. 在完整训练集上重新训练最佳模型
3. 在测试集上生成预测结果
4. 保存为Kaggle提交格式

## 七、实验结论

### 7.1 主要发现

1. **最佳模型**: Gradient Boosting以83.16%的交叉验证准确率脱颖而出，成为泰坦尼克号生存预测的最佳模型。

2. **模型性能排序**:
   - Gradient Boosting (83.16%)
   - AdaBoost (81.82%)
   - Random Forest (80.81%)
   - Logistic Regression (79.69%)
   - Naive Bayes (79.47%)

3. **性能特点**:
   - 集成学习模型（Gradient Boosting、AdaBoost、Random Forest）表现最佳
   - 简单模型中Logistic Regression表现良好
   - Naive Bayes训练速度最快（0.002秒）
   - SVM在本次数据集上表现相对较弱

4. **特征重要性**:
   - 性别是最重要的预测特征
   - 客舱等级和票价反映社会地位影响
   - 家庭规模和称谓提供额外信息

### 7.2 实验成果

1. **预测结果文件**: `Gradient_Boosting_submission.csv`
   - 格式符合Kaggle提交标准
   - 包含418条预测记录

2. **可视化图表**: 生成了多个对比图表，便于直观理解模型性能

3. **性能数据**: `model_comparison_summary.csv` 记录了所有模型的详细性能指标

### 7.3 技术收获

1. 掌握了机器学习项目的完整流程：数据预处理 → 特征工程 → 模型训练 → 评估优化 → 结果预测

2. 学会了多种机器学习算法的应用和对比方法

3. 理解了不同模型的特点和适用场景

4. 掌握了交叉验证等评估方法的使用

### 7.4 不足与改进方向

**当前不足**:
1. 特征工程可以进一步优化
2. 超参数调优不够深入
3. 没有尝试模型集成策略

**改进方向**:
1. **超参数调优**: 使用GridSearchCV或RandomizedSearchCV进行更系统的调参
2. **特征工程**: 尝试更多特征组合和转换
3. **模型集成**: 结合多个模型的预测结果
4. **交叉验证**: 使用更严格的验证策略（如StratifiedKFold）
5. **特征选择**: 使用特征选择方法筛选最优特征子集
